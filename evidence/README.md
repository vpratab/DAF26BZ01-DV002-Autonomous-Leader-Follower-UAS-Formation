# Evidence Guide

The `evidence/` directory contains deterministic outputs generated by `./scripts/run_scenarios.sh`.

## Scenario Inventory

| Scenario | Directory | Purpose |
| --- | --- | --- |
| [Nominal Single-Pilot Formation Leg](./scenario_01_nominal_formation_leg/trust_scorecard.json) | `scenario_01_nominal_formation_leg` | Leader-follower geometry remains stable with healthy timing, navigation coherence, and valid handoff planning. |
| [Contested-PNT Stasis Drift](./scenario_02_degraded_pnt_spacing/trust_scorecard.json) | `scenario_02_degraded_pnt_spacing` | Formation spacing and timing drift begin to accumulate under degraded PNT but remain recoverable under reversion or stasis logic. |
| [Follower Divergence After Handoff](./scenario_03_follower_divergence/trust_scorecard.json) | `scenario_03_follower_divergence` | The follower departs safe spacing and heading coherence while target-handoff validity collapses, driving a reject verdict. |

## Files Per Scenario

- `input_log.jsonl`: scenario input trace used by the runtime monitor
- `timeline.json`: frame-by-frame trust and property outcomes
- `trust_scorecard.json`: summary counts and final trust score
- `proof_log.txt`: concise textual ledger of non-accept events
- `trace.svg`: lightweight trust-over-time visualization
- `summary.json`: compact machine-readable scenario summary

## Recommended Review Path

0. read `scorecard_summary.md` for the cross-scenario result table
1. open `trust_scorecard.json`
2. inspect `timeline.json` for the first non-accept frame
3. inspect `proof_log.txt` for the short event ledger
4. open `trace.svg` for a quick visual read
